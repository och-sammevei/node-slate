    Introduction

    Welcome to the SammeVei API documentations. We use several endpoints, serving different purposes. These will be described in detail further down this document.

    Main database

    Everything is contained in the main database, which is accessed through our Parse server. In this document, we do not go into the specifics of reading or writing data to the Parse server, as this is very well documented at http://parseplatform.org.

    For iOS we are obviously using the Parse iOS SDK, and for Android we'll use the Parse Android SDK.

    Make sure to replace meowmeowmeow with your API key.

    Kittn uses API keys to allow access to the API. You can register a new Kittn API key at our developer portal.

    Kittn expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: meowmeowmeow

    You must replace meowmeowmeow with your personal API key.

    Endpoints

    Database

    We are using Parse with MongoDb as our main storage engine.

    PathFinder

    We have built this route matching API to easily find travels matching a specific route from A to B, with the direction and time filters.

    IO

    Websocket interface for pushing communications between users, for instance when requesting a ride or responding to a request.

    All communication between users have Push notifications as a fallback, in case we were unable to deliver the message when the recipient is offline or have the app in the background.

    Pathfinder

    Welcome to the SammeVei API documentation. There are more than one API endpoints, serving different purposes, like the PathFinder, which finds published travels matching a route.

    The main database is using Parse, so familiarize yourself with how this is accessed on your target platform.

    To use our APIs and the Parse database, several keys are required.

    Journeys

    Get all Journeys

    curl "https://pathfinder.sammevei.no/api/1/journeys"
  -H "X-PFAPI-KEY: <your-pathfinder-api-key>"
  -H "X-SVUID-KEY: <user-id>"

    The above command returns JSON structured like this:

    [
  {
    "id": 1,
    "name": "Fluffums",
    "breed": "calico",
    "fluffiness": 6,
    "cuteness": 7
  },
  {
    "id": 2,
    "name": "Max",
    "breed": "unknown",
    "fluffiness": 5,
    "cuteness": 10
  }
]

    This endpoint retrieves all journeys, with a batch size of 20 per page. When listing journeys, route information is NOT provided.

    HTTP Request

    GET https://pathfinder.sammevei.no/api/1/journeys

    Query Parameters

    Parameter Default Description
    include_cats false If set to true, the result will also include cats.
    available true If set to false, the result will include kittens that have already been adopted.

    Get a specific Journey

    curl "https://pathfinder.sammevei.no/api/1/journey/<ID>"
  -H "X-PFAPI-KEY: <your-pathfinder-api-key>"

    The above command returns JSON structured like this:

    {
  "id": 2,
  "name": "Max",
  "breed": "unknown",
  "fluffiness": 5,
  "cuteness": 10
}

    This endpoint retrieves a specific journey, including it's route information.

    HTTP Request

    GET https://pathfinder.sammevei.no/api/1/journeys/<ID>

    URL Parameters

    Parameter Description
    ID The ID of the Journey to retrieve

    Count active Journeys

    HTTP REQUEST

    curl "https://pathfinder.sammevei.no/api/1/journey/<ID>"
  -H "X-PFAPI-KEY: <your-pathfinder-api-key>"

    The above command should return a JSON structure:

    {
  "count": 33
}

    GET https://pathfinder.sammevei.no/api/1/journeys/count

    limit Limit the number of returned matches. 3

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request sucks
    401 Unauthorized -- Your API key is wrong
    403 Forbidden -- The kitten requested is hidden for administrators only
    404 Not Found -- The specified kitten could not be found
    405 Method Not Allowed -- You tried to access a kitten with an invalid method
    406 Not Acceptable -- You requested a format that isn't json
    410 Gone -- The kitten requested has been removed from our servers
    418 I'm a teapot
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarially offline for maintanance. Please try again later.
